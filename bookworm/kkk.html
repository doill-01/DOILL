제작개요

고객참여 중심으로 변화하는 비즈니스 트렌드에 걸맞는 사이트를 제작하여
고객이 소비자에 국한되지 않고 주체자가 될 수 있는 도서 관련 환경을 구축함.

    제작 순서

  1. 공정도 및 ui 제작

  2. erd작성 및 담당파트 배정

  3. 코드작성

  4. 코드 작성 및 발표

느낀점

프로젝트의 목표는 온라인 서점의 기본적인 사이클과 채팅기능을 수행할 수 있는 사이트를 구현하는 것이었다. 총 4주간의 프로젝트 기간인데 시작 시엔 프로젝트 기본 환경 세팅 및 기획 db구성 품의서등 4일가량 걸려 이걸 과연 할 수 있을까 라는 고민이 컸었다. 하지만 할 수 있는 만큼 최대한 구현해보자는 생각으로 끈기 잇게 하나하나씩 해나 갓고 부족하지만 초기 목표한 기본적인 사이클을 수행할 수 있는 결과물을 만들 수 있었다.  프로젝트를 진행하면서  앞서 약 4개월간 수업에서 배웠던 지식들(java, html,css, 스프링) 직접 적용해보며 내가 직접 체험할 수 있었던 시간이었다. 프로젝트 초반 중반까진 어노테이션 사용법, api 사용 미숙, 데이터 혼동, html, css, javascript 등 모든 것이 미숙하여 일일이 찾아가면서 하느라 시간이 많이 들었다. 하지만 후반에 가면서 기본적인 기능들에 대해서 어느 정도 익숙해져서 데이터를 주고받는 방식을 고민하거나, 혹은 구현하고 싶은 기능이 있으면 해당 기능을 수행하기까지의 순서도만 고민하면 그리 어렵게 않게 만들어 갈 수 있었다.
